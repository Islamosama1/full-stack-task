## Prompts and approaches that worked well

- **"Scaffold X following NestJS conventions"** — giving a target module name (e.g., `AuthModule`) alongside an explicit list of responsibilities (cookie management, JWT strategy, repository pattern) produced clean, idiomatic NestJS code with correct decorator usage in one pass.
- **"Write unit tests for this service; mock all dependencies"** — passing the actual service file as context let the AI infer the exact constructor signature and generate correctly typed Jest mocks without back-and-forth.
- **"Add a global exception filter that returns `{ statusCode, error, message, timestamp, path }`"** — a single, concrete output shape in the prompt meant the filter was correct and required no edits.
- **Incremental commit-by-commit prompting** — keeping each prompt scoped to one concern (cookies, guards, DTOs) produced focused diffs that were easy to review, matching the commit history in this repo.

## What had to be corrected or reworked

- **Linting errors after shadcn scaffolding** — the AI-generated shadcn form components had unused imports and implicit `any` types that the ESLint config flagged. These were fixed in a dedicated `fix: linting error` commit.
- **Cookie-parser integration** — the initial `main.ts` generated by AI was missing the `cookie-parser` middleware registration. It was added in a follow-up commit (`back: add cookies parser`) after runtime testing revealed the cookie was never parsed.
- **`tsconfig.test.json` path aliases** — Jest could not resolve path aliases in the test files because the AI-generated `tsconfig.test.json` was missing the `paths` entries that matched `tsconfig.json`. The fix required a manual edit before `npm run test:unit` passed.
- **Duplicate shadcn component** — the AI added a second signup form component (`src/components/signup-form.tsx`) alongside the one already in `src/components/auth/SignupForm.tsx`. The stale duplicate was left in place but the canonical component under `auth/` is the one rendered by the app.
- **Docker hot-reload** — the first compose file did not mount source volumes into the containers, so code changes were not reflected without rebuilding. Volume mounts were added manually after noticing the issue during development.
